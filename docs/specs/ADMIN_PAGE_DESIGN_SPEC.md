# Admin 페이지 디자인 명세서

이 문서는 Admin 페이지의 UI/UX 디자인을 위한 명세서입니다.  
디자이너가 참고하여 화면 디자인을 진행할 수 있도록 작성되었습니다.

**기반 문서**: `docs/ADMIN_PAGE_REQUIREMENTS.md` (기능 명세서)  
**디자인 시스템**: `docs/02_UI_DESIGN_SYSTEM.md` 참고

---

## Admin 페이지 구조 개요

Admin 페이지는 크게 **2가지 주요 영역**으로 구분됩니다:

### Part 1: 캐릭터 콘텐츠 관리 (Content Management)
캐릭터(춘심, 소라, 미나 등)의 콘텐츠를 관리하는 영역
- 캐릭터 정보 관리 (프로필, 페르소나 프롬프트)
- 사진/동영상 갤러리 관리
- 캐릭터 추가/삭제

### Part 2: 사용자 및 운영 관리 (User & Operations Management)
서비스 운영과 사용자 관리를 위한 영역
- 사용자 관리 및 통계
- 결제/구독 관리
- 대화/메시지 관리
- 시스템 모니터링

## 1. 전체 구조 및 레이아웃

### 1.1 기본 레이아웃 구조

```
┌─────────────────────────────────────────────────────┐
│  Header (상단 고정)                                   │
│  [Logo]  [검색바]  [알림]  [프로필]                    │
├──────────┬──────────────────────────────────────────┤
│          │                                          │
│ Sidebar  │  Main Content Area                       │
│ (고정)   │  (스크롤 가능)                            │
│          │                                          │
│          │                                          │
│          │  Part 1: 캐릭터 콘텐츠 관리                │
│ - 캐릭터 │  - 캐릭터 목록                            │
│   관리   │  - 캐릭터 편집                            │
│          │                                          │
│          │  Part 2: 사용자 및 운영 관리               │
│ - 사용자 │  - 사용자 목록                            │
│   관리   │  - 사용자 통계                            │
│ - 결제/  │  - 결제/구독 통계                         │
│   구독   │                                          │
│ - 대화   │  - 대화방 목록                            │
│   관리   │  - 메시지 검색                            │
│ - 통계/  │  - 대시보드                               │
│   모니터 │  - API 사용량                             │
│ - 콘텐츠 │  - 뉴스/이벤트                            │
│   관리   │  - 미션 관리                              │
│ - 설정   │  - 시스템 설정                            │
│          │                                          │
└──────────┴──────────────────────────────────────────┘
```

### 1.2 레이아웃 특징
- **사이드바**: 왼쪽 고정, 항상 표시 (접기/펼치기 옵션)
  - Part 1과 Part 2를 시각적으로 구분 (구분선 또는 섹션 헤더)
- **헤더**: 상단 고정, 스크롤 시에도 항상 보임
- **메인 콘텐츠**: 반응형, 최소 너비 1200px (데스크톱 최적화)
- **색상 테마**: 다크 모드 우선 또는 라이트/다크 토글
- **Part 구분**: 사이드바 메뉴와 페이지에서 Part 1/Part 2 명확히 구분

- **Admin 프로필**: 오른쪽 (드롭다운: 프로필, 로그아웃)

## 4. 공통 UI 컴포넌트

### 4.1 버튼
- **Primary**: 주요 액션 (저장, 확인 등)
- **Secondary**: 보조 액션 (취소, 닫기 등)
- **Danger**: 삭제, 차단 등 위험한 액션
- **Text**: 텍스트 링크 스타일

### 4.2 입력 필드
- 텍스트 입력
- 텍스트 영역 (멀티라인)
- 드롭다운 (Select)
- 날짜/시간 선택기
- 파일 업로드 (드래그 앤 드롭 지원)

### 4.3 테이블
- 정렬 가능한 헤더
- 행 선택 (체크박스)
- 액션 컬럼 (편집, 삭제 버튼)
- 페이지네이션

### 4.4 모달/다이얼로그
- 확인 다이얼로그 (삭제 확인 등)
- 폼 모달 (빠른 편집)
- 전체 화면 모달 (큰 폼)

### 4.5 차트
- 라인 차트 (추이)
- 바 차트 (비교)
- 파이 차트 (비율)
- 숫자 카드 (통계 요약)

### 4.6 알림
- Toast 알림 (성공/에러)
- 배지 (개수 표시)
- 경고 메시지

## 5. 상태 표시

### 5.1 온라인 상태
- **Online**: 초록색 점 (●)
- **Offline**: 회색 점 (○)

### 5.2 구독 등급
- **FREE**: 회색 배지
- **BASIC**: 파란색 배지
- **PREMIUM**: 보라색 배지
- **ULTIMATE**: 금색 배지

### 5.3 로딩 상태
- 스켈레톤 UI (콘텐츠 로딩)
- 스피너 (액션 로딩)
- 프로그레스 바 (업로드 등)

## 6. 반응형 디자인

### 6.1 브레이크포인트
- **데스크톱**: 1200px 이상 (기본)
- **태블릿**: 768px - 1199px (사이드바 접기)
- **모바일**: 768px 미만 (모바일 메뉴)

### 6.2 모바일 대응
- 사이드바: 햄버거 메뉴로 변경
- 테이블: 카드 형태로 변환 또는 스크롤
- 차트: 모바일 최적화 크기

## 7. 접근성 (Accessibility)

- 키보드 네비게이션 지원
- 스크린 리더 지원 (ARIA 레이블)
- 색상 대비 비율 (WCAG AA 준수)
- 포커스 표시 명확히

## 8. 디자인 시스템 참고

기존 메인 앱의 디자인 시스템을 참고하되, Admin 페이지는 별도 스타일 가능:

- **색상**: 다크 테마 우선 또는 라이트/다크 토글
- **타이포그래피**: 가독성 중시 (데스크톱)
- **아이콘**: Material Symbols 또는 유사 아이콘 시스템
- **간격**: 일관된 간격 시스템 (8px 그리드 권장)

## 9. 우선순위별 디자인 작업 순서
