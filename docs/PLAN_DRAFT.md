# 춘심 챗봇 프로젝트 기획서

## 1. 프로젝트 개요

### 1.1 프로젝트 목적

- **캐릭터**: 춘심 (여자 캐릭터, 아이돌/애인 역할)
- **타겟 사용자**: X(트위터) 팔로워 3만 2천명

- **서비스 형태**: 별도 웹/앱 채팅 플랫폼

- **핵심 가치**: 팔로워들이 춘심과 일상적인 대화를 나누며 감정적 연결을 형성

### 1.2 비전

팔로워들이 춘심과 자연스럽고 따뜻한 대화를 통해 일상의 동반자이자 특별한 존재로 느낄 수 있는 경험 제공

## 2. 춘심 캐릭터 설정

### 2.1 캐릭터 컨셉

- **역할**: 아이돌 + 애인 (팔로워와의 관계에 따라 다르게 표현)

- **성격**: 여러 스타일을 선택/조합 가능

- 귀엽고 밝은 아이돌 스타일

- 로맨틱하고 다정한 애인 스타일
- 두 스타일의 혼합

- **대화 주제**: 기본적으로 일상 대화 중심

### 2.2 캐릭터 일관성 관리

- 춘심의 성격, 말투, 반응 패턴을 일관되게 유지

- 사용자가 선택한 스타일 조합에 맞춰 응답 생성

- 대화 히스토리를 기반으로 관계 발전 추적

## 3. 사용자 경험 (UX) 시나리오

### 3.1 첫 방문 시나리오

1. 사용자가 웹/앱 접속
2. X 트위터 OAuth 로그인

3. 춘심 캐릭터 소개 화면

4. 캐릭터 스타일 선택 (선택사항)

5. 첫 인사 및 대화 시작

### 3.2 일상 대화 시나리오

1. 사용자가 일상적인 이야기를 전달

2. 춘심이 공감하고 응답

3. 자연스러운 대화 흐름 유지

4. 대화 히스토리 자동 저장

### 3.3 재방문 시나리오

1. 로그인 후 이전 대화 히스토리 확인

2. 마지막 대화 이어서 진행

3. 관계 발전 정도 반영된 인사

## 4. 핵심 기능 요구사항

### 4.1 인증 및 사용자 관리

- **X 트위터 OAuth 로그인**

- 팔로워 여부 확인 (선택사항)

- 사용자 프로필 정보 연동

- **사용자 세션 관리**

- 로그인 상태 유지

- 자동 로그아웃 방지

### 4.2 채팅 기능

- **실시간 채팅 인터페이스**

- 메시지 입력 및 전송

- 춘심의 응답 표시

- 로딩 상태 표시

- **대화 히스토리**

- 과거 대화 조회

- 대화 검색 (선택사항)

- 대화 삭제 (선택사항)

### 4.3 캐릭터 커스터마이징

- **스타일 선택/조합**

- 아이돌 스타일, 애인 스타일, 혼합 스타일

- 여러 스타일 동시 활성화 가능

- 스타일별 프롬프트 관리

- **캐릭터 일관성 유지**

- 선택한 스타일에 맞는 응답 생성

- 대화 컨텍스트 기반 일관성 유지

### 4.4 AI 응답 생성

- **Gemini API 연동**
- 사용자 메시지 분석

- 춘심 캐릭터에 맞는 응답 생성
- 선택된 스타일 반영

- **컨텍스트 관리**

- 대화 히스토리 활용

- 관계 발전 정도 반영

- 일상 대화 주제에 맞춘 응답

## 5. 비즈니스 로직

### 5.1 대화 흐름 관리

- 사용자 메시지 → Gemini API → 춘심 응답 생성 → 저장 → 표시

- 대화 히스토리를 컨텍스트로 포함하여 일관성 유지

- 스타일 조합에 따른 프롬프트 조정

### 5.2 데이터 관리

- **대화 데이터 저장**

- 사용자별 대화 히스토리
- 메시지 타임스탬프

- 선택된 캐릭터 스타일 정보

- **사용자 설정 저장**

- 선호하는 캐릭터 스타일

- 알림 설정 (선택사항)

### 5.3 성능 최적화

- 대화 히스토리 페이징 처리

- API 응답 시간 최적화

- 캐싱 전략 (선택사항)

## 6. UI/UX 기획

### 6.1 메인 채팅 화면

- **레이아웃**

- 상단: 춘심 프로필/상태 표시

- 중앙: 대화 메시지 영역 (스크롤 가능)

- 하단: 메시지 입력창

- **디자인 컨셉**

- 따뜻하고 친근한 느낌

- 춘심 캐릭터 이미지 활용

- 모바일 친화적 반응형 디자인

### 6.2 설정 화면

- 캐릭터 스타일 선택/변경

- 알림 설정
- 계정 관리

### 6.3 대화 히스토리 화면

- 과거 대화 목록

- 날짜별/주제별 필터링 (선택사항)

## 7. 데이터베이스 설계 (요약)

### 7.1 주요 테이블

- **users**: 사용자 정보 (X 트위터 연동)

- **conversations**: 대화 세션

- **messages**: 개별 메시지

- **user_settings**: 사용자 설정 (캐릭터 스타일 등)
