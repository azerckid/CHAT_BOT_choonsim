## 🚀 Overview

**ChoonSim-Talk** is a relationship-sustaining AI companion built around a real IP with **33,000+ global fans** on X (Twitter). We combine deep AI companionship with on-chain investment mechanics to introduce a novel **"Raise-to-Earn"** model.

### ⚠️ The Problem
1. **Cold Start in AI Chat**: Existing AI services lack shared history with users, making conversations hollow and engagement shallow.
2. **IP Revenue Black Box**: Revenue generated by character IPs stays locked inside centralized platforms — fans and investors have no transparent way to participate in the IP's growth.

### 💡 Our Solution
1. **5-Layer Context Engine**: ChoonSim carries a living history. Her personality, memories, and emotional identity are ported from X into the app, creating deep 1:1 bonds recorded as on-chain credit data on Creditcoin 3.0 (CC3).
2. **BondBase Revenue Bridge**: Every interaction requiring 'CHOCO' (in-app currency for chat, gifts, memory engraving) generates revenue. This revenue is aggregated per character and sent to [BondBase](https://github.com/azerckid/BLOCKCHAIN_BOND_BASE), where it becomes **on-chain Bond yield**, shared directly with Bond holders.
3. **Character Growth = Investment**: The more users nurture ChoonSim, the higher the Bond value. Fans play to grow the character; investors buy to earn real yield.

## ✨ Key Features

- **Interactive AI Companion**: Real-time chat with streaming responses, powered by Google Gemini and augmented by a multi-layer context engine (LangChain/LangGraph) for sustained emotional memory.
- **On-Chain Settlement (CC3)**: User interactions and IP revenue are anchored to the Creditcoin 3.0 network, ensuring transparent credit history and revenue distribution.
- **BondBase Integration**: Seamless bridging of off-chain AI interactions to on-chain DeFi yields.
- **Fandom Ecosystem**: Daily missions, memory engraving (NFT-like mechanics), and a tiered subscription model (FREE, BASIC, PREMIUM, ULTIMATE).
- **Progressive Web App (PWA)**: Accessible anywhere, designed with a premium, mobile-first UI using shadcn/ui and Tailwind CSS v4.

## 🛠 Tech Stack

### Frontend
- **Framework**: React Router v7 (Vite)
- **UI/UX**: shadcn/ui (Nova Preset), Tailwind CSS v4, Framer Motion
- **State & Data**: React Hooks, Zod (Validation), Luxon (Date/Time)

### Backend & AI
- **Runtime**: Node.js / Vercel Edge
- **Database**: Turso (libSQL) with Prisma ORM (Edge compatible)
- **Auth**: Better Auth (Email, Social: Google/X/Kakao)
- **AI Engine**: Google Gemini API, LangChain / LangGraph (Context & Memory management)
- **Media**: Cloudinary

### Web3 Integration
- **Blockchain**: Creditcoin 3.0 (CC3)
- **DeFi Protocol**: BondBase (On-chain revenue tokenization)

## 🏁 Getting Started (Local Development)

### Prerequisites
- Node.js 20+
- Turso Database account & token
- Google Gemini API Key
- Cloudinary credentials
- Better Auth secret

### Installation

1. **Clone the repository**
   ```bash
   git clone https://github.com/azerckid/CHAT_BOT_choonsim.git
   cd CHAT_BOT_choonsim
   ```

2. **Install dependencies**
   ```bash
   npm install
   ```

3. **Environment Setup**
   Create a `.env` file based on `.env.example` (you will need to provide your own API keys for the app to function fully).
   ```env
   # Database (Turso)
   TURSO_DATABASE_URL=
   TURSO_AUTH_TOKEN=
   
   # Auth
   BETTER_AUTH_SECRET=
   BETTER_AUTH_URL=http://localhost:5173
   
   # AI
   GEMINI_API_KEY=
   
   # Media
   CLOUDINARY_CLOUD_NAME=
   CLOUDINARY_API_KEY=
   CLOUDINARY_API_SECRET=
   ```

4. **Database Migration**
   ```bash
   npx prisma generate
   npx prisma migrate dev
   ```

5. **Run Development Server**
   ```bash
   npm run dev
   ```
   The app will be available at `http://localhost:5173`.
